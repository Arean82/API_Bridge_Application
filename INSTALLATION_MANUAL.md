# API Bridge Application – Simple Installation Manual

## 1️⃣ Install from Source

1. **Clone the repository**
   ```bash
   git clone https://github.com/Arean82/API_Bridge_Application.git
   cd API_Bridge_Application
   ```
2. **Create a virtual environment** (isolates dependencies)
   ```bash
   # Windows PowerShell
   python -m venv venv
   .\venv\Scripts\Activate.ps1   # or .\venv\Scripts\activate.bat

   # Linux / macOS
   python3 -m venv venv
   source venv/bin/activate
   ```
3. **Install required Python packages**
   ```bash
   pip install -r requirements.txt
   ```

   This installs Flask, Flask‑Cors, OpenTelemetry, SQLAlchemy and all other libraries the app needs.

You now have a working development copy of the API Bridge ready for the next step.

---

## 2️⃣ Build with PyInstaller

The repository ships a helper script `build.py` that wraps PyInstaller.

1. **Run the build helper**

   ```bash
   python build.py
   ```

   You will see a menu:
   ```
   === Universal API Bridge Build System ===
   What would you like to build?
     1. OneDir  (folder with many files – fastest build)
     2. OneFile (single .exe – clean distribution)
     3. Both
     4. Exit
   ```
2. **Choose the build mode**

   - **OneDir** – Recommended for a quick first build. PyInstaller creates a `dist/onedir/` folder containing `bridge_app.exe` (or a Linux binary) and all required DLLs/shared objects.
   - **OneFile** – Produces a single standalone executable (`dist/onefile/bridge_app.exe`). This takes longer and may require extra DLLs, but results in a single file you can copy elsewhere.
3. **What PyInstaller does behind the scenes**

   - Collects **all Python modules** imported by the project (`bridge_app/` package).
   - Detects **runtime data files** (e.g., `config.ini`, static assets) and bundles them.
   - Includes **hidden imports** needed for Flask and OpenTelemetry.
   - Packs the Python interpreter itself, so the result runs on any compatible OS without a separate Python install.
4. **After the build completes**

   - The executable will be located in `dist/onedir/` (OneDir) or `dist/onefile/bridge_app.exe` (OneFile).
   - Test it by running:
     ```bash
     # Windows
     dist\onedir\bridge_app.exe

     # Linux / macOS
     ./dist/onedir/bridge_app
     ```
   - You should see the message:
     ```
     === API Bridge started ===
     Running on http://0.0.0.0:5000/ (Press CTRL+C to quit)
     ```
   - Open a browser at `http://localhost:5000/` to confirm the API endpoint returns JSON.

That’s all you need to get the application built and runnable. You can now distribute the executable or run it directly.

## 3️⃣ Production Deployment

### Prerequisites

- A Linux server (Ubuntu 22.04+ recommended) or Windows Server.
- Root/administrator access.
- Nginx installed (or any reverse‑proxy of your choice).
- Python 3.10+ installed.

### 1. Clone the repository and install dependencies

```bash
git clone https://github.com/Arean82/API_Bridge_Application.git
cd API_Bridge_Application
python -m venv venv
# Windows PowerShell
.\\venv\\Scripts\\Activate.ps1   # or .\\venv\\Scripts\\activate.bat
# Linux / macOS
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure for production

Edit `config.ini`:

- Set `debug = False`
- Set `environment = production`
- Adjust `host` if you need to bind to a specific IP.

### 3. Create a systemd service (Linux)

Create `/etc/systemd/system/api-bridge.service`:

```ini
[Unit]
Description=API Bridge Application
After=network.target

[Service]
WorkingDirectory=/path/to/API_Bridge_Application
ExecStart=/path/to/API_Bridge_Application/venv/bin/python run.py
Restart=always
User=www-data
Group=www-data
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

Enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable api-bridge.service
sudo systemctl start api-bridge.service
```

### 4. Set up Nginx as a reverse proxy

Create `/etc/nginx/sites-available/api_bridge`:

```nginx
server {
    listen 80;
    server_name your.domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Enable the site and reload Nginx:

```bash
sudo ln -s /etc/nginx/sites-available/api_bridge /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### 5. Verify deployment

```bash
curl -I http://your.domain.com
```

You should receive a `200 OK` response with JSON from the API.

## 4️⃣ Production Deployment from PyInstaller Binary

### Prerequisites

- The compiled executable (`bridge_app.exe` on Windows or `bridge_app` on Linux) generated by the PyInstaller build step.
- A target server (Windows or Linux) where you will run the binary.

### 1. Transfer the binary

Copy the executable to the target host, e.g.:

```bash
scp dist/onedir/bridge_app user@server:/opt/api-bridge/
# Windows example using WinSCP or copy the .exe manually
```

### 2. Ensure the config file is present

Place a `config.ini` file in the same directory as the binary and edit it for production:

- `debug = False`
- `environment = production`
- Adjust `host` as needed.

### 3. Run the binary

```bash
cd /opt/api-bridge/
./bridge_app   # Windows: bridge_app.exe
```

The application will start using the host/port defined in `config.ini`.

### 4. (Optional) Run as a systemd service (Linux)

Create `/etc/systemd/system/api-bridge-binary.service`:

```ini
[Unit]
Description=API Bridge (PyInstaller binary)
After=network.target

[Service]
WorkingDirectory=/opt/api-bridge
ExecStart=/opt/api-bridge/bridge_app
Restart=always
User=www-data
Group=www-data
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

Enable and start the service:

```bash
sudo systemctl daemon-reload
sudo systemctl enable api-bridge-binary.service
sudo systemctl start api-bridge-binary.service
```

### 5. Set up Nginx (full deployment)

#### Install Nginx (if not already installed)

- **Ubuntu/Debian:**

  ```bash
  sudo apt update
  sudo apt install nginx -y
  sudo systemctl enable --now nginx
  ```
- **RHEL/CentOS/Fedora:**

  ```bash
  sudo dnf install nginx -y
  sudo systemctl enable --now nginx
  ```

#### Configure Nginx as a reverse proxy

Create a server block file `/etc/nginx/sites-available/api_bridge` (or `/etc/nginx/conf.d/api_bridge.conf`):

```nginx
server {
    listen 80;
    server_name your.domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

Enable the site and reload Nginx:

```bash
sudo ln -s /etc/nginx/sites-available/api_bridge /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

#### Verify Nginx is proxying correctly

```bash
curl -I http://your.domain.com
```

You should receive a `200 OK` response and the JSON from the API.
